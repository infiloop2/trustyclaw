Turn repository requests into tested, reviewed pull requests on connected GitHub repositories.

What it does

Inspects a repository, implements a focused change, runs verification, opens the pull request, and follows review through a quiet green round.

Tools it uses

GitHub is required. Brave Search is optional for current technical research.

Background routines

None by default. The agent can schedule a one-shot CI or review follow-up when useful.

Quality bar

Definition of ready

A pull request is ready only when the requested outcome is complete and GitHub confirms the result.

  1. The focused implementation, tests, and documentation are committed.
  2. Relevant local verification and GitHub checks pass.
  3. Every review comment has a fix or reasoned reply.
  4. The full diff is free of stale and incidental changes.
You keep the merge decision.
